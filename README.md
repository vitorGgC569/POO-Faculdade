## Avisos

### 1. Introdução e Objetivo
Esse repositorio foi criado para armazenamento de códigos em sala, tudo que está aqui é conforme solicitado pelo professor. Não é e não contém as melhores práticas de programação.
Devido a origem do código e intenção didática de sala de aula. Se busca códigos bons, ou completos, esse não é o repositorio ideal.

### 2. Modelagem (Diagrama de Classes)
Na pasta assets de cada projeto modelado haverá o diagrama de classes passado para a codificação em aula e seguindo as etapas do professor, interfaces, heranças e demais tecnologias,
serão somente utilizadas quando apresentadas em sala e na etapa correspondente.
